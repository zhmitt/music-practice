# Capability Spec: Task Registry

## Purpose

Provide a deterministic registry view over current OpenSpec changes.

## Current State

- `workflow/state/task-registry.md` is generated by `workflow/scripts/tasks-sync.sh`.
- The registry summarizes active, draft, ready-to-archive, and archived changes.
- The registry is canonical runtime state and no longer mirrors `.specify/memory/task-registry.md`.

## Requirements

### Requirement: Deterministic change summary

The task registry MUST reflect the current `openspec/changes/` tree.

#### Scenario: syncing registry state

- **WHEN** `workflow/scripts/tasks-sync.sh` runs
- **THEN** it regenerates `workflow/state/task-registry.md` from canonical change files
- **AND** it exposes the next step for each non-archived change

### Requirement: Staleness detection

The registry MUST support a deterministic check mode for hooks and quality gates.

#### Scenario: checking staged tasks

- **WHEN** `workflow/scripts/tasks-sync.sh --check` runs
- **THEN** it exits non-zero if `workflow/state/task-registry.md` is stale
- **AND** the pre-commit hook can block the commit until the registry is refreshed
