# /user-test - Automatic User Testing

Executes user acceptance tests via Claude in Chrome and returns **raw observations** for Main Agent (Opus) analysis.

## Usage

```bash
/user-test --critical      # P0 critical tests only (~5 min)
/user-test --regression    # All P0+P1 tests (~15 min)
/user-test --feature 024   # Feature-specific tests
```

## Architecture: Observer + Analyst Split

```
User Test Runner (Sonnet)     Main Agent (Opus)
─────────────────────────     ─────────────────
Execute test steps      →     Analyze observations
Capture screenshots     →     Determine root causes
Record console errors   →     Assess deploy readiness
Record network failures →     Recommend fixes
Report exact DOM state  →     Correlate with code
```

**Why:** Sonnet is fast and cheap for browser automation. Opus has full project context for accurate analysis.

## What It Does

1. **Detects user-facing features**
2. **Starts Claude in Chrome session**
3. **Executes test scripts** (generated by `/speckit.usertest`)
4. **Returns raw observations** (screenshots, errors, DOM state, timing)

The **Main Agent (Opus)** then:
- Analyzes the observations
- Identifies root causes (correlating with code knowledge)
- Judges deploy readiness
- Recommends specific fixes

## Test Modes

### Critical (`--critical`)
- P0 tests only, ~5 minutes
- Use after implementing critical features

### Regression (`--regression`)
- All P0 + P1 tests, ~15-20 minutes
- Use after deployment to Dev

### Feature-Specific (`--feature {id}`)
- Tests for one feature only, ~3-5 minutes

## When to Use

**Use for:**
- Critical user flows (auth, core features)
- Regression testing after deployment
- Smoke tests after changes

**Don't use for:**
- Non-user-facing features (APIs, background jobs)
- First-time feature exploration (use manual)
- Rapid iteration phase

## Integration

### With Post-Implementation

```
/post-impl
  ├─ /test (technical tests)
  ├─ Generate user test scripts
  ├─ /docs (documentation)
  └─ /user-test --critical
      └─ Returns observations → Main Agent analyzes
```

## Requirements

- Claude in Chrome installed
- User test scripts generated (`/speckit.usertest`)
- Dev server running
- Feature is user-facing
