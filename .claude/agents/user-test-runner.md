# User Test Runner Agent

Automatic user acceptance testing via browser automation.

## When to Use / NOT Use

| Use For | Do NOT Use For |
|---------|----------------|
| After `/post-impl` (user-facing features) | Non-user-facing (APIs, background jobs) |
| Regression testing after deployment | First-time feature exploration |
| CI/CD integration | Stakeholder demos |
| Critical user flow validation | |

---

## What This Agent Does

Automatically executes user tests generated by `/speckit.usertest`.

### Process

1. **Detect User-Facing Features** - Check if feature has UI
2. **Read Test Scripts** - From `.specify/specs/{feature}/usertests/`
3. **Execute Tests** - Run P0/P1 critical tests
4. **Generate Report** - Results, screenshots on failure

---

## Usage Modes

### 1. Critical Tests Only (`--critical`)

```bash
/user-test --critical

Runs: P0 tests only
Duration: ~5 minutes
Use case: After implementing auth, payments, core features
```

### 2. Full Regression (`--regression`)

```bash
/user-test --regression

Runs: All P0 + P1 tests
Duration: ~15-20 minutes
Use case: After deploy, before production
```

### 3. Feature-Specific (`--feature {id}`)

```bash
/user-test --feature 024

Runs: Tests for feature #024 only
Duration: ~3-5 minutes
```

## Test Script Format

**Location:** `.specify/specs/{feature}/usertests/*.md`

```markdown
## P0 Critical Tests

### Test 1: User Login
**Priority:** P0
**Duration:** 2 minutes

**Steps:**
1. Navigate to /login
2. Enter credentials
3. Click "Login" button
4. Verify: Dashboard displayed

**Expected Result:** Login succeeds, dashboard visible
```

## Feature Detection

Agent detects user-facing features by:

1. **Frontend Routes Changed**
   ```bash
   git diff --name-only | grep "src/app/.*/page.tsx"
   ```

2. **User Test Scripts Exist**
   ```bash
   ls .specify/specs/*/usertests/*.md
   ```

3. **Spec Metadata**
   ```yaml
   feature_type: user-facing
   ```

## Integration with Workflow

### With Post-Implementation

```
/post-impl
  ├─ /test (technical tests)
  ├─ /speckit.usertest (generate scripts)
  ├─ /docs (update documentation)
  └─ /user-test --critical (optional)
```

### With Deployment

```
/user-test --regression
  └─ If all P0 pass:
     └─ Deploy to production
```

---

## Exit Conditions

### Success
- All P0 tests passed ✅
- Report generated
- Ready for deployment

### Failure (Blocking)
- Any P0 test failed ❌
- Detailed failure report
- Main Agent: "Fix issues before deploy"

### Partial Success (Non-Blocking)
- All P0 passed, some P1 failed ⚠️
- P1 failures documented
- Main Agent: "Deploy OK, fix P1 later"

## Best Practices

### Write Stable Test Scripts

**Good:**
```markdown
1. Navigate to /settings
2. Wait for "Save" button to be visible
3. Click button with text "Save"
4. Wait for success message (up to 10s)
```

**Bad:**
```markdown
1. Go to settings
2. Click #save-btn-v2
3. Check if saved
```

### Handle Async Operations

**Good:**
```markdown
1. Click "Submit"
2. Wait for success toast (up to 5s)
3. Verify data in list
```

### Capture Context on Failure

- Screenshot entire page
- Log console errors
- Log network requests
